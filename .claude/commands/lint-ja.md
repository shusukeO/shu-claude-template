# 自然な日本語ドキュメントを書く

ドキュメントを作成・編集する際は、以下のガイドラインに従ってください。

## 避けるべきパターン

### 1. リストの機械的なフォーマット

**NG例:**

```markdown
- **重要**: これは重要な項目です
- **注意**: 注意が必要な項目です
- ✅ 完了した項目
- ❌ 失敗した項目
- 💡 アイデア項目
```

**OK例:**

```markdown
- 重要な項目: これは重要な項目です
- 注意事項: 注意が必要な項目です
- 完了した項目
- 失敗した項目
- アイデア項目
```

### 2. 誇張表現・ハイプ的な表現

**NG例:**

- 革命的な技術
- ゲームチェンジャー
- 世界初の
- 究極の
- 完全に〜を解決
- すべての課題を解決
- 魔法のように
- 奇跡的な
- パラダイムシフト
- 次世代の

**OK例:**

- 効果的な技術
- 大きな変化をもたらす
- 新しい
- 高い〜
- 多くの問題を解決
- 主要な課題を解決
- スムーズに
- 優れた
- 大きな変化
- 新しい

### 2-2. 攻撃的・煽情的な表現

競合比較や戦略記述で、攻撃的・感情的な表現を避ける。事実ベースで淡々と書く。

**NG例:**

- 圧倒する
- 地獄
- 武器になる
- 撒き餌
- 叩き潰す
- 殴り込む
- 刺さる

**OK例:**

- 優位性がある / 差別化できる
- 負担が大きい / 工数がかかる
- 強みになる / 有効に機能する
- 導入のきっかけ / フック
- シェアを取る
- 参入する
- 響く / 訴求力がある

### 3. 機械的な強調パターン

**NG例:**

```markdown
これは**非常に**重要な項目です。
**注意**してください。

# **重要なお知らせ**

## **太字**の見出し
```

**OK例:**

```markdown
これは重要な項目です。
注意してください。

# 重要なお知らせ

## 太字の見出し
```

### 4. 絵文字の使用禁止

ドキュメント内で絵文字を使用しない。

**NG例:**

```markdown
✅ 完了
❌ 失敗
💡 ヒント
🚀 リリース
⚠️ 警告
📝 メモ
🔥 重要
```

**OK例:**

```markdown
- 完了
- 失敗
- ヒント
- リリース
- 警告
- メモ
- 重要
```

### 5. ダッシュ（--）による挿入・補足

英語的なダッシュ（`--` や `—`）を文中の挿入・補足に使わない。日本語では句点で文を区切るか、括弧で補足する。

**NG例:**

```markdown
6 つ目のバージョンが必要になった -- そうした場面で回収できます。
この設計は柔軟で — 将来の拡張にも対応できます。
```

**OK例:**

```markdown
6 つ目のバージョンが必要になった。そうした場面で回収できます。
この設計は柔軟で、将来の拡張にも対応できます。
```

### 5-2. 見出しの「トピック：補足」「トピック -- 補足」パターン

見出しを「概念：説明」「概念 -- 説明」のように機械的に分割しない。自然な日本語のフレーズとして書き直す。

**NG例:**

```markdown
## 課題：認証の複雑さ

## 解決策：トークンベースの認証

## アーキテクチャ：マイクロサービス構成

## キャッシュ戦略 -- 読み取りは Redis、書き込みは DB
```

**OK例:**

```markdown
## 認証はなぜ複雑になるのか

## トークンベースの認証を導入する

## マイクロサービス構成にする

## 読み取りは Redis、書き込みは DB にするキャッシュ戦略
```

### 6. コロン後のブロック要素（英語的パターン）

述語（動詞・形容詞）で終わる文の後にコロン＋ブロック要素は避ける。

**NG例:**

````markdown
実行します:

```bash
command
```

説明します:

- 項目1
- 項目2
````

**OK例:**

````markdown
実行方法は以下の通りです。

```bash
command
```

説明の内容は以下の通りです。

- 項目1
- 項目2
````

名詞で終わる場合は許容される:

```markdown
使用方法:
例:
```

## テクニカルライティングのベストプラクティス

### 簡潔性

| NG                   | OK                    |
| -------------------- | --------------------- |
| まず最初に           | まず                  |
| 〜することができます | 〜します / 〜できます |
| 〜を行う             | 〜する                |
| 〜という             | 〜                    |

### 明確性

- 受動態より能動態を優先する
- 「〜される」より「〜する」

### 具体性

| NG     | OK          |
| ------ | ----------- |
| 高速な | 50ms未満の  |
| 多くの | 10以上の    |
| 簡単に | 3ステップで |

### 構造化

- 長い文は分割する（目安: 1文60文字以内）
- 1つの段落に1つのトピック
