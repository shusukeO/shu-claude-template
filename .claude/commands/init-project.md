# プロジェクト初期化

新しいプロジェクトでClaude Codeを使い始めるための初期設定を行います。

## Prompt

以下の手順でプロジェクトを初期化してください。

### 1. プロジェクト構造の把握

プロジェクトのディレクトリ構造を確認し、以下を特定してください。

- ルートディレクトリの構成
- 主要なソースコードディレクトリ
- 設定ファイル（package.json, Cargo.toml, pyproject.toml, go.mod 等）
- ドキュメントディレクトリ
- テストディレクトリ

### 2. 技術スタックの確認

設定ファイルから以下を特定してください。

- プログラミング言語とバージョン
- フレームワーク
- 主要な依存関係
- ビルドツール
- テストフレームワーク
- リンター/フォーマッター

### 3. 開発コマンドの確認

以下のコマンドを特定してください。

- 依存関係のインストール
- 開発サーバーの起動
- ビルド
- テストの実行
- リント/フォーマット

### 4. CLAUDE.md の作成

上記の情報を基に、CLAUDE.md を作成してください。以下のセクションを含めます。

```markdown
# CLAUDE.md

## Development Commands

（開発コマンド一覧）

## Architecture Overview

（ディレクトリ構造）

## Tech Stack

（技術スタック）

## Development Rules

（開発ルール・注意点）
```

### 5. ドキュメントディレクトリの作成

docs ディレクトリが存在しない場合は、以下の構造で作成してください。

```
docs/
├── todo/       # 未完了タスク・計画
│   └── .gitkeep
└── done/       # 完了タスク・アーカイブ
    └── .gitkeep
```

### 6. 初期化完了の報告

初期化が完了したら、以下の情報をユーザーに報告してください。

- 作成/更新したファイル一覧
- 検出した技術スタックの概要
- 次のステップの提案（例: MCPサーバーの設定、カスタムコマンドの追加等）

## 注意事項

- 既存のCLAUDE.mdがある場合は、上書き前にユーザーに確認を求める
- プロジェクト固有の設定やルールがある場合は、それらを尊重する
- 不明な点がある場合は、ユーザーに確認を取る
