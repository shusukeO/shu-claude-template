# ドキュメントと実装の同期チェック

以下のドキュメントと実際の実装を比較し、差異があれば修正してください。

## チェック対象

1. CLAUDE.md（存在する場合）
   - 技術スタック（設定ファイルのバージョンと一致しているか）
   - ディレクトリ構造（実際のプロジェクト構造と一致しているか）
   - 主要なファイル・コンポーネント一覧
   - 関連リソースのリンク

2. README.md（存在する場合）
   - 技術スタック
   - プロジェクト構造
   - ドキュメントセクション

3. docs/配下のドキュメント（存在する場合）
   - 各ドキュメントの内容が現状と一致しているか

## 確認手順

1. プロジェクトのディレクトリ構造を再帰的に取得
2. 設定ファイル（package.json, Cargo.toml, pyproject.toml等）のバージョン情報を確認
3. ドキュメントディレクトリの内容を確認
4. 各ドキュメントと比較し、差異を特定
5. 差異があれば修正

## 修正方針

- 空ディレクトリは「将来の拡張用」と明記するか、不要であれば記載しない
- 新規追加されたファイル・コンポーネントはドキュメントに追加
- 削除されたファイル・ディレクトリはドキュメントから削除
- バージョン番号はメジャーバージョンのみ記載（例: Python 3.12, Node.js 20）
